### Understanding the Credit System

**Message credits** are spent when you prompt the AI to build or change your app. Every message — "add a login button", "fix this bug", "add a payment form" — costs credits. Iterating on a feature costs credits each time.

**Integration credits** are spent when your app's users take actions that involve integrations: LLM calls, file uploads, email sends, database queries. Each costs 1 credit regardless of type.

**The critical thing to know:** credits don't roll over. Unused credits expire at the end of your billing cycle. You also can't buy extra credits without upgrading your entire plan — which has frustrated some users mid-project.

## Where Base44 Falls Short

### 1. Credits run out faster than expected
The most consistent complaint across Reddit, ProductHunt, and our community. During active development — especially debugging — message credits evaporate quickly. Every re-prompt, every fix attempt, every "try this instead" costs credits. Users who build complex apps often burn through their monthly allocation in a few sessions.

**The workaround:** Use discussion mode to plan thoroughly before prompting. Edit code directly (available from Starter) instead of re-prompting for small fixes.

### 2. Complexity wall on advanced logic
Base44 handles CRUD apps, dashboards, portals, and booking systems extremely well. Where it struggles: complex conditional logic, multi-step workflows with many dependencies, and very custom backend behavior. The AI can get stuck in loops where each fix creates a new issue.

**The workaround:** Break complex features into smaller, isolated prompts. Build incrementally rather than describing a complex system in one shot.

### 3. Login page branding
Base44 shows its own branding on login and signup screens. You can't fully customize this experience on any plan. For customer-facing products where brand consistency matters, this is a real limitation.

### 4. No backend export
Frontend code exports to GitHub from the Builder plan. But your backend logic and database stay on Base44's infrastructure permanently. If you ever need to migrate away, you're rebuilding the backend from scratch.

### 5. Support response times
Paid users on Reddit have reported slow support response times, particularly for blocking issues. For solo builders this is manageable. For teams shipping on a deadline, it introduces risk.
